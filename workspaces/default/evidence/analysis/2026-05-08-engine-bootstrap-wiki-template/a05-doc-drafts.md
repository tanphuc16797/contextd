## DRAFT-G2 — wiki-usage.md  (group: Per-task pipeline)

> File path: `{ws}/projects/engine/services/wiki-usage.md`
> Covers: E-006..E-008 (3 commands) — main user-facing flow

### Purpose
3 commands chính cho task wiki-aware: planning before code (`/contextd-use`), syncing wiki sau code change (`/contextd-update`), full re-validation (`/contextd-rebase`).

### Entry-points

#### `/contextd-use`
- **Inputs**: task description (free-text)
- **Outputs**: `.claude/context/current-task.md`, `.claude/runs/{run_id}/` traces, code generated by Builder
- **Flow** (5 stages — pattern P-007):
  1. Stage 0: workspace resolve (P-001)
  2. Stage 1: `contextd-planner` → intent JSON
  3. Stage 2: `contextd-context-selector` → write current-task.md
  4. Stage 2.5: `contextd-plan-reviewer` → APPROVED/BLOCK
  5. Stage 3: Builder (main agent) → code
  6. Stage 4: `contextd-reviewer` (optional) → violations
- **Calls**: A-001, A-002, A-003, A-005 (sub-agents); `templates/prompt-template.md`
- **Citation**: `(.claude/commands/contextd-use.md:L1)`, `(agents/pipeline/README.md:L27-L46)`
- **Failure handling**:
  - Stage 2.5 BLOCK → STOP, không tiếp tục Stage 3
  - Stage 4 violations → output report, KHÔNG sửa code

#### `/contextd-update`
- **Inputs**: optional `--scope`
- **Outputs**: edits in `{ws}/...`
- **Flow**: detect git diff → delegate to `contextd-curator` (A-004)
- **Citation**: `(.claude/commands/contextd-update.md:L1)`

#### `/contextd-rebase`
- **Inputs**: none
- **Outputs**: edits in `{ws}/...`
- **Flow**: scan wiki vs codebase → delegate to `contextd-curator` (A-004)
- **Citation**: `(.claude/commands/contextd-rebase.md:L1)`

## DRAFT-G3 — codebase-analysis.md  (group: Bootstrap wiki từ source code)

> File path: `{ws}/projects/engine/services/codebase-analysis.md`
> Covers: E-009 (1 command, but central to evidence pipeline)

### Purpose
Snapshot metadata codebase (variant=code or agentic-engine) → ingest vào evidence pipeline → auto-trigger analysis.

### Entry-points

#### `/code-analyze`
- **Inputs**: `--ref {path}`, `--scope {globs}`, `--bundle {dir}`, `--label`, `--skip-analyze`, `--allow-configs`, `--with-drafts`, `--variant {code|agentic-engine}`
- **Outputs**:
  - `{ws}/evidence/sources/{evid-id}/{raw.md, source.yaml, raw.normalized.md?}`
  - Auto-trigger `/evidence-analyze` (unless `--skip-analyze`)
- **Flow**:
  1. Workspace resolve (P-001)
  2. Variant detect (P-008): markers count, validation gate, scope defaults
  3. evid-id derive: `{date}-{code|engine|platform}-{slug}`
  4. Confirm preview (P-002)
  5. Build raw.md per variant template (10 sections)
  6. Redaction post-pass (P-004)
  7. source.yaml + sha256 + dedup check
  8. _index.md row + state=ingested
  9. Trigger `/evidence-analyze` (variant-specific CORE set)
- **Calls**: templates `code-snapshot.md` / `agentic-engine-snapshot.md`, `evidence-source.yaml`; command `/evidence-analyze`
- **Citation**: `(.claude/commands/code-analyze.md:L1, L43-L70, L194-L297)`
- **Failure handling**:
  - Repo not recognized → STOP với gợi ý `--variant`
  - Working tree dirty → AskUserQuestion (Continue/Abort)
  - Duplicate sha256 → STOP, reuse existing evid-id
  - Secret detected post-redaction → STOP `SECRET DETECTED IN SNAPSHOT`

### Patterns applied
- P-001, P-002, P-003 (config guard), P-004 (redaction), P-008 (variant dispatcher)

## DRAFT-G6 — evidence-pipeline.md  (group: External raw → wiki)

> File path: `{ws}/projects/engine/services/evidence-pipeline.md`
> Covers: E-014..E-018 (5 commands) — central evidence flow

### Purpose
4-stage state machine pipeline (ingest → analyze → qa → apply) đưa raw data từ MCP/API/paste/code vào wiki structured docs.

### State machine
Per pattern P-005 / contract C-005:
```
ingested → analyzed → qa_in_progress ⇄ qa_awaiting_external → qa_done → applied → archived
```

### Entry-points

#### `/evidence-ingest`
- **Inputs**: `--source {mcp|api|paste|code}`, `--label`, source-specific args
- **Outputs**: `{ws}/evidence/sources/{evid-id}/{raw.*, source.yaml}`; `_index.md` row state=`ingested`
- **Citation**: `(.claude/commands/evidence-ingest.md:L1)`

#### `/obsidian-ingest`
- **Inputs**: `--vault`, `--folder`
- **Outputs**: batch ingests via `/evidence-ingest --source paste`
- **Citation**: `(.claude/commands/obsidian-ingest.md:L1)`

#### `/evidence-analyze`
- **Inputs**: `--id`, `--prompt {NN | cNN | aNN}`
- **Outputs**: `{ws}/evidence/analysis/{id}/*.md`; transition `ingested → analyzed`
- **Dispatch** (per P-008):
  - `source_type ∈ {paste, api, mcp}` → CORE `[01, 02, 04, 08]`
  - `source_type=code, code_variant=code` → CORE-CODE `[c01, c02, c03, c04, 04, 08]`
  - `source_type=code, code_variant=agentic-engine` → CORE-AGENTIC `[a01, a02, a03, a04, 04, 08]`
- **Citation**: `(.claude/commands/evidence-analyze.md:L1)`

#### `/evidence-qa`
- **Inputs**: `--id`
- **Outputs**: `{ws}/evidence/qa/{id}/{recommendations.md, batches/, verified-facts.md, pending-external.md?}`; transition `analyzed → qa_in_progress → qa_done`
- **Flow**: C8 QA Recommender pre-batch-1 (P0/P1 only) → user batches → verified-facts.md
- **Citation**: `(.claude/commands/evidence-qa.md:L1)`, `(agents/pipeline/qa-batching.md:L1)`

#### `/evidence-apply`
- **Inputs**: `--id`, `--mode {update|rebase}`, `--dry-run`
- **Outputs**: edits in `{ws}/{platform,domains,projects,decisions,runbooks}/...`; transition `qa_done → applied`
- **Calls**: `contextd-curator` (A-004); templates `service.md`, `pattern.md`, `adr.md`, `runbook.md`
- **Citation**: `(.claude/commands/evidence-apply.md:L1)`
